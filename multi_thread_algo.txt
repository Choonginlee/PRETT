multi thread 알고리즘

1. bfs
- 스레드 (기본 5개) 생성
- 노드마다 스레드 개별적으로 할당
- 다음 레벨로 가면 또 할당 할당 할당
- token수가 유한하므로, max depth를 정해 제한을 둠
- max depth는 우선 5 정도?

2. dfs
- 각 노드마다 스레드 생성
- 가장 처음으로 노드 찾은 스레드 생기면 -> 다 멈춰?
- 하나 찾아서 들어감 -> 스레드 쭉 생성 -> 반복
- 스레드 다 돌면? -> 부모노드로 돌아와서 스레드 재시작 (..흠;;)
- 끝날때까지 반복


prune 알고리즘

1. bfs
- 1단계 다 찾으면 -> 2단계 각 노드에 다시 토큰 다 쏴보고, 1단계 결과랑 같은지 비교 -> 같으면 없애고, 다르면 남김 (시간 얼마나 오래걸리려나..)
2. dfs
- 하나 찾으면 -> 들어가서 찾았을때 까지 쏴본거 다시 쏴봄 (맞나 이게) -> 똑같으면, 찾은게 아님..? -> 부모노드로 돌아가서 다시 시작
